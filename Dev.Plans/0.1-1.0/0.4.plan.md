# Version 0.4 Plan

**Version:** 0.4 — **Codename:** Expansion

---

## Overview

Version 0.4 focuses on expanding the calculator suite by adding new STEM domains beyond chemistry. This release introduces a Physics Calculator module with core mechanics and thermodynamics tools, expanding HelpOS into a multi-discipline problem-solving engine.

---

## Goals

- Launch Physics Calculator as a new primary domain
- Implement 3-4 essential physics calculators with deterministic accuracy
- Maintain consistent UI/UX patterns from 0.3
- Use existing architecture without major refactors

---

## Key Features / Workstreams

### Physics Engine (Primary)

- **Kinematics Calculator**
  - Solve: v = u + at, s = ut + ½at², v² = u² + 2as
  - Support variable solving (find any value given others)
  - Display formulas clearly with proper superscript/subscript notation

- **Force & Motion Calculator**
  - F = ma calculations
  - Weight vs. mass conversions (g = 9.81 m/s²)
  - Net force with multiple forces

- **Projectile Motion Calculator**
  - Time of flight, range, max height
  - Support custom angles and initial velocities
  - Display trajectory info clearly

- **Thermodynamics Calculator**
  - Q = mcΔT (heat capacity)
  - Energy unit conversions (J, cal, kJ)
  - Phase change calculations (latent heat)

### Shared Physics Validation

- Create centralized physics constants module (matching CAST standards where applicable)
- Define: gravitational acceleration, gas constant, Boltzmann constant, etc.
- Use consistent precision across all physics calculators

### UI Integration

- Add "Physics" tab/section to calculator menu
- Reuse existing formula rendering from 0.3
- Show version "0.4: Expansion" in UI
- Ensure consistent input/output styling

### Testing

- Add unit tests for each physics calculator
- Validate against known textbook examples
- Test edge cases (zero values, negative inputs where invalid)

---

## Acceptance Criteria

- Physics Calculator section is accessible and functional
- All 4 calculators produce accurate results for standard problems
- Version displays as "0.4: Expansion" throughout UI
- Formula notation renders correctly (superscripts for exponents, units clear)
- No regressions in Chemistry or Math calculators from 0.3

---

## Timeline

- **30 min:** Set up physics engine module, constants, validation test cases
- **20 min:** Implement kinematics and force calculators
- **10 min:** Add projectile motion and thermodynamics calculators
- **Remaining:** UI integration, testing, version branding

---

## Risks & Mitigations

- Risk: Physics formulas have more edge cases (division by zero, impossible scenarios)
  - Mitigation: Add input validation and user-friendly error messages
- Risk: Thermodynamics unit conversions add complexity
  - Mitigation: Use existing unit conversion utilities; test thoroughly

---

## Not Included (v0.4)

- Quantum mechanics
- Optics
- Waves & sound
- Electromagnetism (defer to v0.5)
