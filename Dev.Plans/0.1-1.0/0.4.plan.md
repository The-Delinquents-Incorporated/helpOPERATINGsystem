## Key Features / Workstreams

### AI-Assisted Chemistry Solvers (Primary)

Each solver integrates with Ollama to explain work with a user-controlled toggle.

#### Stoichiometry Solver

- **Input:** Chemical equation + given value (mass, moles, volume, particles)
- **Output:** Any stoichiometric answer (other masses, moles, volumes, particles)
- **AI Integration:** Ollama explains step-by-step solution
- **Toggle:** "Show Work"
  - ON: AI provides detailed explanation in response
  - OFF: AI calculates and returns only the answer
- **Use Case:** Student enters equation + "25 g of NaCl" → solver returns all possible conversions

#### Empirical & Molecular Formula Solver

- **Input:** Percent composition or experimental data + molar mass (if known)
- **Output:** Empirical formula and molecular formula
- **AI Integration:** Ollama explains derivation process
- **Toggle:** "Show Work"
  - ON: AI walks through steps (find moles → lowest ratio → molecular formula)
  - OFF: AI returns formulas only
- **Use Case:** "Given 40% C, 6.7% H, 53.3% O and M = 180" → C₆H₁₂O₆

#### Percent Composition Solver

- **Input:** Chemical formula
- **Output:** Percent composition by mass for each element
- **AI Integration:** Ollama explains calculation method
- **Toggle:** "Show Work"
  - ON: Shows molar mass calc, element mass, percentage math
  - OFF: Returns composition table only
- **Use Case:** "Find % composition of H₂SO₄" → H: 2.04%, S: 32.69%, O: 65.25%

#### Percent Error Solver

- **Input:** Theoretical value + experimental value
- **Output:** Percent error with sign and magnitude
- **AI Integration:** Ollama explains accuracy interpretation
- **Toggle:** "Show Work"
  - ON: Shows formula, calculation, interpretation (high error = ?, low error = ?)
  - OFF: Returns percent error only
- **Use Case:** "Theoretical: 18.02 g/mol, Experimental: 17.8 g/mol" → 1.22% error

### Shared AI Architecture

- All solvers use the same Ollama prompt template with work explanation control
- Prompt injection should prevent AI from performing raw math (keep calculations deterministic)
- Each solver validates its output against deterministic engine before returning

## Risks & Mitigations

- Risk: AI sometimes makes calculation errors instead of explaining
  - Mitigation: Validate all AI numeric outputs against deterministic engine before display
- Risk: Knowledge base becomes too large and search is slow
  - Mitigation: Use client-side indexing or simple string matching for offline search
- Risk: User toggles work on/off but gets inconsistent formatting
  - Mitigation: Define strict output format in Ollama prompt template
