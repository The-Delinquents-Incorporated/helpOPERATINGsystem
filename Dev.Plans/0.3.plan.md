## 2. Calculator Input Formatting & LaTeX Translation

Both calculators should support natural scientific typing while displaying professionally formatted notation. User-friendly input should automatically be translated into the appropriate internal representation for calculation.

### Chemistry Calculator

Support chemical formulas written using underscore notation.

**Examples**

| User Types | Displayed As | Internal Interpretation |
|------------|--------------|-------------------------|
| `H_2O` | H₂O | Two hydrogen atoms and one oxygen atom |
| `CO_2` | CO₂ | Carbon dioxide |
| `NH_4Cl` | NH₄Cl | Ammonium chloride |
| `Ca(OH)_2` | Ca(OH)₂ | Calcium hydroxide |

Requirements:

- `_` indicates the following number is a chemical subscript.
- Render subscripts using proper typography throughout the interface.
- Parse the displayed formula into the chemistry engine for all calculations.
- Preserve correct stoichiometric interpretation internally.
- The displayed formatting must never interfere with calculation accuracy.

### Math Calculator

Support exponent notation using the caret (`^`) operator.

**Examples**

| User Types | Displayed As | Internal Interpretation |
|------------|--------------|-------------------------|
| `x^2` | x² | x raised to the second power |
| `3^4` | 3⁴ | 3 to the fourth power |
| `(x+1)^3` | (x + 1)³ | Cubic expression |
| `10^-6` | 10⁻⁶ | Scientific notation |

Requirements:

- `^` represents exponent notation.
- Render exponents using superscript typography or LaTeX rendering.
- Translate exponent expressions into the backend's mathematical evaluation engine.
- Support nested expressions, parentheses, and complex formulas.
- Display mathematical expressions exactly as they would appear in textbooks while preserving computational correctness.

### Unified Rendering Pipeline

Both calculators should use the same formatting architecture:

1. User enters natural scientific notation.
2. Input is parsed into an abstract syntax representation.
3. The interface renders polished mathematical/chemical notation.
4. The parsed representation is sent to the calculation engine.
5. Results are displayed using the same high-quality formatting.

This ensures users can type naturally while the application simultaneously:

- Displays professionally formatted notation.
- Correctly interprets scientific meaning.
- Performs accurate calculations.
- Maintains consistency across all calculators.

### Chemistry Engine Validation & Accuracy Fixes

The chemistry calculation engine must be audited and validated to ensure every calculator produces scientifically accurate results. Although the official atomic masses already exist within the codebase, the application is currently producing incorrect outputs, indicating an issue with formula parsing, data retrieval, or calculation logic rather than missing reference data.

#### Current Issue

A validation test revealed an incorrect result:

| Input | Calculator | Expected | Current |
|--------|------------|---------:|--------:|
| `Cl` | Molar Mass | **35.45 g/mol** | **100 g/mol** |

This demonstrates that the calculator is not correctly interpreting or processing the existing atomic mass data.

#### Scope

This validation must apply to **every Chemistry Calculator**, including but not limited to:

- Molar Mass
- Percent Composition
- Empirical Formula
- Molecular Formula
- Stoichiometry
- Limiting Reactant
- Percent Yield
- Solution Chemistry
- Gas Laws
- Dilution
- Any future chemistry calculation modules

The fix must be implemented at the shared chemistry engine level rather than patching individual calculators.

#### Required Investigation

The implementation should verify:

- Correct parsing of chemical formulas.
- Correct identification of each element.
- Correct retrieval of official atomic masses.
- Correct handling of subscripts and element counts.
- Correct multiplication by atom quantity.
- Correct summation of total molar mass.
- Correct unit conversions where applicable.
- Correct display formatting after calculation.

If any discrepancy exists between the parser, internal representation, and calculation engine, it must be resolved before release.

Every chemistry calculator should be tested against a comprehensive suite of known compounds using the official California Department of Education (CAST) periodic table values already stored within the project.
